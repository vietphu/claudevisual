Chạy & Debug ClaudeVisual trong VS Code

Giải thích mô hình "Extension Development Host" — vì sao nhấn F5 không tự động là "chạy extension", và cửa sổ nào mới là nơi cần nhìn.

1 — Vấn đề đang gặp
Bối cảnh

VS Code extension không chạy trực tiếp trong cửa sổ bạn đang gõ code. Nó cần một cửa sổ VS Code thứ hai, gọi là Extension Development Host, đóng vai trò "phòng thí nghiệm" để thử extension mà không ảnh hưởng cửa sổ chính. Nhấn F5 chỉ là lệnh "mở phòng thí nghiệm đó lên" — nếu thiếu cấu hình hoặc phòng thí nghiệm mở trống, bạn sẽ không thấy gì cả.

3 — F5 thực sự làm gì (theo thứ tự)
  1. VS Code đọc .vscode/launch.json

    File này mô tả "chạy cái gì". Nếu chưa có file này, F5 không biết phải làm gì — đây là lý do lần đầu bạn nhấn F5 "không thấy gì" (tôi vừa tạo file này).

  2. Chạy task build trước (preLaunchTask)

    Task npm: compile chạy tsc (kiểm tra kiểu dữ liệu) rồi esbuild (đóng gói code thành dist/extension.js). Nếu bước này lỗi, cửa sổ mới sẽ không mở.

  3. Mở cửa sổ "Extension Development Host"

    Đây là một tiến trình VS Code hoàn toàn mới, riêng biệt — không phải cửa sổ bạn đang thao tác. Tiêu đề cửa sổ này thường có chữ [Extension Development Host].

  4. Extension host nạp dist/extension.js và gọi activate()

    Đây là lúc code trong src/extension.ts thực sự chạy — nhưng chỉ trong cửa sổ mới này.

  5. Extension bắt đầu tail JSONL và vẽ status bar

    Nếu đúng folder ClaudeVisual đang mở trong cửa sổ mới, status bar góc dưới trái sẽ hiện model/token đang dùng.

4 — Hai cửa sổ, hai vai trò khác nhau
CỬA SỔ GỐC — nơi bạn gõ lệnh này
  • Chứa source code (src/*.ts)
  • Nơi bạn nhấn F5 để khởi động
  • Có thanh Debug (nút Continue/Step/Stop) khi đang debug
  • Extension KHÔNG chạy ở đây — đây chỉ là nơi điều khiển
EXTENSION DEVELOPMENT HOST — cửa sổ tự mở ra
  • Một cửa sổ VS Code hoàn toàn khác, tự bật lên sau F5
  • Extension ClaudeVisual thực sự chạy ở đây
  • Phải mở đúng folder ClaudeVisual ở đây thì mới thấy gì
  • Status bar cần xem nằm ở góc dưới trái của cửa sổ này
claude-sonnet-5 · ~12% · 45.2k tok
Dễ nhầm nhất
Sau khi nhấn F5, bạn vẫn đang nhìn vào cửa sổ gốc theo phản xạ — nhưng status bar cần xem nằm ở cửa sổ thứ hai vừa bật lên. Hãy tìm cửa sổ có tiêu đề chứa [Extension Development Host].
